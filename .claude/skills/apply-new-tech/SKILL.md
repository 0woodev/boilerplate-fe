다음 기술을 이 FE 프로젝트에 도입하려고 한다:

**도입 대상:** $ARGUMENTS

---

## 작업 순서

### 1. 현재 프로젝트 구조 파악

아래 파일들을 읽어 현재 구조를 파악한다:

- `package.json` — 현재 의존성 및 스크립트
- `vite.config.js` — 빌드 설정
- `src/` 구조 훑기

핵심 확인 포인트:
- 기존에 유사한 라이브러리가 이미 있는가? (예: 상태관리, 라우팅, 폼)
- 번들 크기 영향은 어느 정도인가?
- CSR/SSR/SSG 중 어떤 렌더링 전략과 맞는가?
- React 18 / Vite 6 호환성 이슈가 있는가?

### 2. 도입 방법 탐색

웹 검색 또는 지식 기반으로 다음을 조사한다:

- 공식 문서 확인 (설치, 기본 설정)
- React + Vite 환경 통합 패턴
- 대안 라이브러리 목록 (최소 2~3개 옵션)
- 번들 사이즈, tree-shaking 지원 여부
- TypeScript 지원 여부

### 3. 도입 방법 비교

아래 기준으로 옵션을 비교한다:

| 기준 | 설명 |
|---|---|
| 번들 크기 | gzip 기준 추가 용량 |
| DX | 개발 경험, 타입 지원, 문서 품질 |
| 유지보수 | npm 주간 다운로드, 마지막 릴리즈 |
| React 통합 | Hooks, Suspense, concurrent mode 호환 |
| 확장성 | 필요 기능만 선택적으로 사용 가능한가 |

### 4. 추천 방향 결정

비교 결과를 바탕으로 **가장 적합한 방법 1개를 추천**한다.

추천 이유:
- 이 프로젝트 구조에 맞는 이유
- 장·단점 요약
- 리스크 및 주의사항

### 5. 구체적 실행 계획 제시

```
STEP 1. 패키지 설치
  npm install {패키지명}

STEP 2. 설정 파일 생성 위치 및 내용

STEP 3. 첫 사용 예시 코드 (React 컴포넌트 기준)

STEP 4. vite.config.js 변경이 필요하다면 변경 내용

STEP 5. 검증 방법 (npm run dev)
```

실제 코드 스니펫을 최대한 포함한다.

### 6. PROGRESS.md 업데이트 제안

도입 결정이 내려지면 `PROGRESS.md`에 추가할 항목을 제안한다:

```markdown
- [ ] **{기술명} 도입** — {한 줄 설명}
```

## 출력 형식

1. **현재 구조 요약** — 기술 도입 전 상태
2. **도입 옵션 비교표**
3. **추천 방향 + 이유**
4. **실행 계획 (단계별 코드 포함)**
5. **PROGRESS.md 업데이트 제안**
